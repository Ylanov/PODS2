# pods2 — мост СЭД (`sed-bridge`)

Браузерное расширение для Яндекс.Браузера / Chrome. Раз в N минут читает разделы СЭД (`sed.mchs.ru`) под уже залогиненной cookie-сессией пользователя и шлёт компактный дайджест в pods2 на эндпоинт `POST /api/v1/sed/snapshot`. На стороне pods2 у юзера в шапке появляется кнопка «Почта · СЭД» с бейджем непрочитанных и панелью со списком писем.

**Что расширение делает:**
- Каждые 5 минут (настраивается) ходит на `https://sed.mchs.ru/decision/delegate`, `/decision/sign`, `/reg/incoming` и т.д.
- Парсит таблицу писем регулярками — без DOM, без зависимостей.
- Если СЭД отдал страницу логина — тихо ждёт, пока пользователь сам войдёт.
- Шлёт собранный JSON в pods2 с заголовком `Authorization: Bearer <token>`.

**Что расширение НЕ делает:**
- Не хранит пароли от СЭД. Пользователь логинится в браузере как обычно, расширение пользуется существующими cookies.
- Не копирует тела документов или файлы. В дайджесте — только заголовок, ID, имя файла и его URL (скачивает потом сам пользователь).
- Не делает ничего, если у юзера в pods2 нет permission'а `sed_inbox`.

## Установка

1. Скачайте/склонируйте папку `extension/sed-bridge/`.
2. В Яндекс.Браузере откройте `browser://extensions/`, включите «Режим разработчика».
3. Нажмите «Загрузить распакованное» и выберите папку `sed-bridge`.

## Настройка

1. Кликните по иконке расширения → «Настройки» (или сразу `browser://extensions/` → «Подробнее» → «Параметры расширения»).
2. **URL pods2** — например `https://staff.asy-tk.ru` (без `/` на конце).
3. **Токен pods2 (Bearer)** — JWT-токен текущего пользователя в pods2.
   - Залогиньтесь в pods2.
   - F12 → Application → Local Storage → ключ `token` → скопируйте значение.
   - Вставьте сюда. Расширение хранит его только в `chrome.storage.local` своего профиля.
4. (Опционально) подправьте список разделов и периодичность.
5. «Сохранить» → «Проверить синхронизацию».

## Что показывает popup

Клик по иконке открывает мини-панель: статус (ok / нужен логин в СЭД / не настроен токен и т.п.), общее число непрочитанных, время последней синхронизации, кнопка «Синхр. сейчас».

## Где смотреть результат на pods2

В шапке pods2 появится кнопка с конвертом — там разделы со счётчиками и заголовки писем. Файлы открываются в новой вкладке на `sed.mchs.ru` — скачивает их браузер пользователя сам, в МЧС-сети.

## Файлы

| Файл | Назначение |
|---|---|
| `manifest.json` | MV3-манифест: permissions, host_permissions, action, alarms, storage |
| `background.js` | Service worker: alarm раз в N минут, fetch разделов, POST в pods2 |
| `parser.js` | Регулярки для извлечения заголовков, файлов и счётчиков из HTML СЭД |
| `options.html` / `options.js` | Страница настроек (URL pods2 + токен + список разделов + периодичность) |
| `popup.html` / `popup.js` | Кнопка-индикатор в тулбаре браузера |
| `icons/` | Иконки 16/48/128 (заглушки — фирменная зелень с белым конвертом) |

## Известные ограничения

- Если СЭД сменит вёрстку, регулярки перестанут парсить. Стабильные точки опоры: `<tr data-entity-id="N">`, `<a href="/node/N">`, `<a href="https://sed.mchs.ru/systems3/...">` — пока они на месте, всё работает.
- Service worker MV3 не имеет постоянной памяти — alarm'ы и хранилище остаются, обычные `let`-переменные в `background.js` сбрасываются. Вся персистентность — через `chrome.storage.local`.
- Если в pods2 сменили токен (logout/login пересоздаёт его), расширение начнёт получать 401 — popup это покажет, надо обновить токен в настройках.

## Безопасность

- Токен pods2 хранится в `chrome.storage.local` (не синхронизируется в облако Яндекса).
- Расширение запрашивает только два домена: `sed.mchs.ru` (читать дайджест) и pods2 (отправлять). Никаких других fetch'ей.
- Cookies СЭД остаются у пользователя — расширение к ним прямого доступа не получает, только пользуется ими через `credentials: 'include'`.

## Проверка работы (после установки)

Чек-лист, чтобы убедиться, что мост СЭД↔pods2 действительно работает.

### 1. На pods2

1. Залогиньтесь в pods2 под админом (или под пользователем, которому собираетесь дать СЭД).
2. Если это **не** админ — выдайте этому пользователю permission `sed_inbox` через вкладку «Пользователи» → «Настроить вкладки».

### 2. Получите токен pods2

1. Залогиньтесь в pods2 под нужным пользователем.
2. F12 → вкладка **Application** (Chrome / Yandex) → **Local Storage** → `https://staff.asy-tk.ru` → ключ `token`.
3. Скопируйте значение целиком (это длинная строка вида `eyJhbGc…`).

### 3. Настройте расширение

1. Кликните на иконку расширения в тулбаре → **Настройки**
   (или `browser://extensions/` → «pods2 — мост СЭД» → «Подробнее» → «Параметры расширения»).
2. **URL pods2**: `https://staff.asy-tk.ru` — без `/` на конце.
3. **Токен pods2**: вставьте из шага 2.
4. **Сохранить**.

### 4. Залогиньтесь в СЭД

В том же браузере откройте https://sed.mchs.ru и залогиньтесь как обычно.
Расширение использует **уже существующую cookie-сессию** — никаких паролей оно не хранит и не запрашивает.

### 5. Запустите синхронизацию вручную

1. Кликните на иконку расширения → **Синхр. сейчас**.
2. В popup'е появится статус. Возможные варианты:
   - ✅ **«Дайджест отправлен. Всего непрочитанных: N»** — всё работает.
   - ⚠️ **«Не залогинены в СЭД»** — откройте sed.mchs.ru и войдите.
   - ⚠️ **«pods2 ответил 401 / 403»** — токен неверный или истёк. Перелогиньтесь в pods2 и обновите токен.
   - ⚠️ **«Не настроен URL pods2 или токен»** — заполните настройки.
   - ❌ **«pods2 недоступен»** — проверьте URL и сетевую связность машины.
3. Параллельно на иконке расширения должен появиться красный бейдж с числом непрочитанных.

### 6. Проверьте на pods2

1. Откройте pods2 в той же вкладке (или обновите страницу).
2. В шапке справа от уведомлений должна появиться иконка **«Почта · СЭД»** с бейджем.
3. Клик по ней — выпадающая панель со списком разделов («На рассмотрение», «На подписание», …) и заголовками писем.
4. Клик по заголовку → новая вкладка с СЭД-документом.
5. Клик по `📎` файлу → новая вкладка → файл скачивается через сессию пользователя в СЭД.

### 7. Что дальше

Дальше расширение само обновляет дайджест каждые 5 минут (можно поменять в настройках). Когда счётчики меняются в СЭД, через минуту-две они отразятся в pods2 — без перезагрузки страницы (через WebSocket).

### Если что-то не работает

- **F12 → Application → Service Workers** → найдите `pods2 — мост СЭД` → клик по ссылке. Откроются логи service worker'а: ищите строки `[sed-bridge]`.
- **`browser://extensions/`** → «Просмотреть представление: service worker» — то же самое.
- В popup всегда виден текущий статус и время последней синхронизации.
